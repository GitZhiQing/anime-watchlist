# Anime Watchlist

中文名：追番计划

一个基于 Bangumi API 的追番软件。

这里的番包括：漫画、动画，在 API 参数中对应 `SubjectType=1`（书籍）& `SubjectType=2`（动画）。

应用并不展示从 Bangumi 获取的全部数据，仅展示必要的。

## 1. 技术栈

使用 Tauri 构建的桌面应用，只支持 Windows 平台。样式基于 shadcn/ui，数据存储于 SQLite。

## 2 整体布局

侧边栏 + 内容区

### 2.1 侧边栏

- 追番（首页）
- 收藏
- 配置

### 2.2 内容区

顶部标题，下方其余空间展示内容。

## 3 页面布局

### 3.1 追番（首页）

顶部标题：追番

从上至下依次展示 在看/想看/看过/搁置/抛弃 五个收藏夹。

这五个栏目可以点击栏目标题实现折叠和展开。

展开状态下，列出该栏目所有番：

```
在看                                         >
封面 | 信息（从上到下：标题、简介、放送时间、标签）
封面 | 信息（从上到下：标题、简介、放送时间、标签）
封面 | 信息（从上到下：标题、简介、放送时间、标签）
...
```

### 3.2 收藏

顶部标题：收藏

```
[漫画/动画名称] [查找]
```

用户输入名字，点击查找，应用会基于 BGM 的 `POST /v0/search/subjects 条目搜索` API 显示所有漫画（`SubjectType=1`）、动画（`SubjectType=2`）。

然后在下方列出一个查找列表，用户可收藏其中的一个或多个条目。

```
封面 | 信息（从上到下：标题、简介、标签、放送时间） | [+: 一个下拉列表，默认显示为收藏，点击展开收藏夹列表，选择对应收藏夹完成收藏]
```

### 3.3 配置

#### 3.3.1 未认证

显示一个 Bangumi 认证按钮，点击后，根据(官方文档)[docs\BGM-API\docs-raw\How-to-Auth.md]中定义的 OAuth 2.0 认证流程完成认证。

#### 3.3.2 已认证

显示用户头像，昵称，用户名，签名，和一个注销按钮。
